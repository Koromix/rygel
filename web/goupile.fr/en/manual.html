Develop pages

Page edition

The page editor helps you develop the form. It is equivalent to a code editor with the difference that here you don't need advanced computer knowledge to develop your form: you can use predefined functions. These functions will generate a visual control in the form: a text field, a numeric field, a date field, a slider, etc.

Code your page in the editor (left), and see the result in the overview (right)

Each change you make in the editor is run and the form overview gives you the result instantly. This is the same page that the final user will see and at this step, you can already enter data.

A page is made of several widgets which allow the user to input data. Many types of widgets are available: text field, number field, etc. To add a widget, you need to call a function. For a complete list and instructions, go to the widget table.

Each widget takes a few options. For input widgets, you need to specify the variable identifier and the label that is displayed to the user. Many other optional settings are available, such as: mandatory, wide, compact, etc.

The code is Javascript, a widely used programming language on the web platform. As a result, you have access to all Javascript features. We will use this later to implement conditional fields and loops, among other things.

Standard widgets

Example of input widgets generated by predefined functions

Widgets are created by calling predefined functions with the following syntax:

function ( "variable_name", "Label shown to user" )

Variable names must start with a letter or _, followed by zero or more alphanumeric or _ characters. They must not contain spaces, accentuated or any other special characters.

Most widgets accept optional arguments with the following syntax:

function ( "variable_name", "Label shown to user", { option1 : value, option2 : value } )

Be careful with code syntax. When parentheses or quotes are mismatched, an error occurs and the page cannot be displayed unless the error is fixed. Go to the section about programming errors for more information.

Input widgets

Widget Code
page.text("variable", "Label")
page.number("variable", "Label")
page.slider("variable", "Label")
page.date("variable_name", "Label")
page.binary("variable_name", "Label")
page.enum("variable_name", "Label", [ ["modality1", "modality1 Label"], ["modality2", "modality2 Label"] ])
page.enumDrop("variable_name", "Label", [ ["modality1", "modality1 Label"], ["modality2", "modality2 Label"] ])
page.enumRadio("variable_name", "Label", [ ["modality1", "modality1 Label"], ["modality2", "modality2 Label"] ])
page.multi("variable_name", "Label", [ ["modality1", "modality1 Label"], ["modality2", "modality2 Label"] ])
page.multiCheck("variable_name", "Label", [ ["modality1", "modality1 Label"], ["modality2", "modality2 Label"] ])
page.calc("variable_name", 18)

Routine use

After the form has been finalized, the form may be used conjointly with the data overview module and the dashboard to check records and progress of data entry.

Dashboard

The dashboard displays the list of records with their status (completed, in progress, empty). From the dashboard, you can access to a specific record in order to complete it or check its values.

Data view and export

Test the form. Immediately check the result with the data overview.

The data overview module displays the data. It also provides an export function of the whole dataset. Currently, supported formats are csv and xlsx.
